# 📊 Amazon India Revenue Leakage Analysis

## 🔍 Overview

This project analyzes **revenue leakage in Amazon India** using a dataset of 10,000+ transactions.

Revenue leakage refers to financial loss occurring after a sale due to:

* Refunds and return abuse
* Operational inefficiencies
* Payment-related issues

## 📈 Key Insights

* Revenue leakage is approximately **~3% of total GMV**
* Around **8% of orders are affected**
* **Prepaid transactions (UPI, Cards)** contribute most to leakage
* **Electronics category** shows highest financial loss
* Leakage is mainly driven by **return and refund issues**

## 🎯 Key Takeaway

Even a small leakage rate (~3%) can lead to **significant financial loss at scale**.
Improving return verification and refund controls can reduce leakage effectively.
